# LICENSE Recommendation Note

## Recommended default

For this repository type (academic/research software with broad reuse goals), **MIT License** is a strong default and is already in use in this project.

### Why MIT is suitable here

- **Low-friction reuse:** maximizes adoption in research, teaching, and prototyping contexts.
- **Compatibility:** works well with mixed open-source ecosystems and downstream tooling.
- **Simplicity:** concise terms that are easy for contributors and reviewers to understand.

## When to choose alternatives

- **BSD-3-Clause** may be preferable if you want MIT-like permissiveness with an explicit non-endorsement clause commonly used in academic labs.
- **Apache-2.0** may be preferable if explicit patent grants/termination terms are important for organizational or commercial collaborations.

## Practical guidance

- Keep one canonical `LICENSE` file in repository root.
- Ensure `CITATION.cff`, README, and release notes consistently reference the selected license.
- Revisit licensing if dependencies, institutional policy, or commercialization plans materially change.
